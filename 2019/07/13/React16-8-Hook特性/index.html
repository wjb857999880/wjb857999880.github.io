React16.8 Hook特性

Posted on 2019-07-13

React 16.8

  • React Hooks发布，可以让你在不编写 class 的情况下使用 state 以及其他的 React 特性
  • 不建议重写现有的应用程序，马上使用Hook，建议尝试在一些新组件中使用Hooks

Hook

添加Hook的原因
  1. 在组件之间复用状态逻辑很难。
  • 缺少复用机制
  • 渲染属性和告诫组件导致层级冗余

    使用 Hook 从组件中提取状态逻辑，使得这些逻辑可以单独测试并复用。

  1. 复杂组件趋向复杂难以维护。
  • 生命周期函数混杂不相干逻辑
  • 相干逻辑分散在不同生命周期

    Hook 将组件中相互关联的部分拆分成更小的函数（比如设置订阅或请求数据），而并非强制按照生命周期划分。

  1. 难以理解的this
  • 函数过度创建新句柄
  • 不能忘记绑定实例

    函数组件无this问题。

    this介绍：把原型方法改变为实例方法，并且强制指定这个方法中的this指向当前的实例，然后该方法才能访问到绑定实例的其它属性。

什么是Hook

  • Hook 是一些可以让你在函数组件里“钩入” React state 及生命周期等特性的函数。
  • Hook 不能在 class 组件中使用。

State Hook

通过在函数组件里调用userState来给组件添加一些内部state，React 会在重复渲染时保留这个 state。

  • 调用useState：会定义一个新的state变量，定义多个则调用多次。
  • useState的参数：state的初始值。
  • userState的返回：当前state以及更新state的函数，这个函数和this.setState类似，但是会替换旧的state，而不是合并。

Effect Hook

当你调用 useEffect 时，就是在告诉 React 在完成对 DOM 的更改后运行设置的 effect 函数。
默认情况下，React 会在每次渲染后调用 effect 函数 —— 包括第一次渲染的时候。

  • useEffect做了什么：告诉React组件需要在渲染后执行某些操作。
  • 为什么在组件内部调用useEffect：可以直接访问state变量或其它props。
  • useEffect会在每次渲染后都执行吗？：React保证了每次运行 effect 时，DOM都已经更新完毕，不用再去考虑挂载还是更新。

effect 函数还可以通过返回一个函数来指定如何清除它。

  • 为什么要在 effect 返回一个函数：可选的清除机制，可以将添加和移除订阅的逻辑放在一起。

  • 何时清除 effect：在组件卸载的时候执行清除操作，会在每次执行当前effect前对上一个effect进行清除。可以通过以下方式避免频繁执行 effect。

Hook使用规则

  1. 只能在函数最外层调用 Hook。不要在循环、条件判断或者子函数中调用。
  2. 只能在 React 的函数组件中调用 Hook。不要在其他 JavaScript 函数中调用。（还有一个地方可以调用 Hook —— 就是自定义的 Hook 中

自定义Hook

当我们想在两个函数之间共享逻辑时，可以提取到第三个函数。

自定义 Hook 是一个函数，其名称以 “use” 开头，函数内部可以调用其他的 Hook。
